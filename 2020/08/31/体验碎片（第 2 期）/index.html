体验碎片（第 2 期）

2020-08-31

Contents

  1. 体验亮点
  2. 体验槽点
  3. 产品期待

为了不局限于技术，更加深刻理解用户侧需求，提升产品体验敏感度。所以会把「体验碎片」系列作为零碎信息专辑，为系统知识构建提供内容。后续会不定期发布。

体验亮点

「即刻」App 给自己动态点赞，在顶部导航会给一个坏笑emoji，以及图像效果

即刻重新上线，对社区内容我不太会花时间经营和阅读。但是对之前信息追踪，包括微博追踪、淘宝店铺上新追踪很有兴趣。不过目前已经没有这些追踪功能。将持续关注后续产品方向。不过发现在给自己发布的动态点赞时，会显示一个坏笑emoji。

以及图像移动动效 + 抖动触感（手机会震动）。这些恶趣味，可以成为社区内容独有特性。

嗯，微信也在 06/17 上线了“拍一拍”功能。

「小宇宙」App 播客内容导入优先级，以及根据时间段评论的产品设计

小宇宙 App 也是即刻团队在 2020/03 上架的一款播客产品，目前应该还在测试阶段（注册还需要邀请码）。因为页面简洁、没有广告。所以最近一直和喜马拉雅交替使用（部分播客内容，小宇宙上并没有）。在这款 App 上，发现有 3 点很有意思。

第 1 点是前期内测版本，没有以小程序为载体。我想除了团队并不缺少研发费用之外，更重要的是播客场景验证、以及播客产品核心价值明确。说直接点，主要还是看内容质量和数量。有了不输于行业内容，再把交互体验做好，很容易脱颖而出。

这点也和小宇宙产品经理有讨论，他认为目前在微信消费播客内容并不是最自然的，用户心智还没有建立。这是一个偏感性的结论，不过我也找到相关数据可以支撑这个结论（下图数据确实可以看到，内容音频这种偏小众，也不能算音乐），确实这种音频内容消费本就不是大众产品，外加小程序体验比不上 App（比如：在小程序内做倍数播放功能，技术实现就比客户端复杂） ，再加上即刻恢复上线，本身就有社区推广渠道。

第 2 点是播客内容生产环节，放弃自己录音，现阶段完全靠其他平台导入。和网易云音乐在前期导入用户歌单如出一撤。内容平台前期，通过这种方式引导内容主过来，平台负责灌流量。这种方式不仅出现在 C 端产品，在电商平台上，新兴平台/小平台商家一直是用这种方式去采集淘宝等大平台商品，去平台分发商品信息。这种设计投入产出高，可谓四两拨千斤。

第 3 点是评论交互细节。可以根据听到某一刻时间节点进行评论。阅读评论点击时间点，播客内容会跳转到对应节点。默认不选中。这个设计抓的场景是“你听他这段，说得很有道理”，可以快速定位到重点位置。

在小宇宙 App 中还有几个细节的产品设计，例如听到嗨处可以点赞形成音浪等细节，也很有意思。

关于播客产品细节设计，少数派有一期邀请王俊煜的节目（_一派·Podcast：024 竖起时间线，原来播客也是可以看的_）。里面有提到轻芒团队播客产品（轻芒团队有做一款类似播客 WordPress 的产品）会有一个设计，就是截取部分音频内容，自动生成带字幕的视频内容，方便用户传播分享。个人觉得这个设计会非常惊艳（已经内测可以体验）。

支付宝语音收钱机盒的切入点

经常有机会看到有线下小商铺不是使用支付宝/微信官方二维码贴纸或者塑料牌，而是使用一种电子屏幕显示的二维码，看起来基本没什么区别。由于好奇，于是去问小卖部阿姨，她的理由是：不用打开支付宝，声音大。

回来在网上查了下这种产品，主打功能就是大喇叭，价格在50.00-100.00 元，月销量也还行。商品详情页宣传的功能点主要是声音：

  1. 老板不在店里，也能播报:添加店员收款通知即可，店员只负责接收通知，付款到老板账户，老板在外还可实时监控收款数据；
  2. 支持多处扫码:拥挤的地方二维码除了贴在收款器上面，也可以贴在其他地方，多处扫码，都能播报；
  3. 专业收款提示芯片：无延迟、声音大、音质清晰。

了解下来还是非常震撼。即使支付宝也做了语音播报。但还是有一个细分市场，在不需要打开支付宝情况下，有宏亮的语音播报，就可以支撑一款 100.00 元的产品。

「抖音」和「快手」视频切换设计逻辑

先分别说下下快手 App 和抖音 App 用户评论时的交互流程。快手双列瀑布流，点击视频直接下拉就是评论区，而刷下一个视频需要退出 + 点击下一个视频两步，并且在视频播放时，“关注”和“分享”的按钮仍在引导。抖音直接下滑就刷新视频，再点击评论，多出 2 步操作。

透过这个设计，来与各自平台出现的现象交叉验证：

  1. 快手以创作者为中心，流量分配规则会照顾个人（相对有人文情怀）；目前全球最大视频库（2019年视频数量 70 亿）；粉丝质量高，黏性高，带货能力强；
  2. 抖音以内容为中心，按照热度和喜好精准推荐；内容生产者粉丝差距大，粉丝集中在大 V。

抖音也知道有黏性的粉丝和活跃的社交互动有价值。但它选消费内容的效率。

所以，疯投圈播客在聊快手 App 时，说到快手社交互动是抖音 3 倍（社交互动定义：（评论数 + 私信）/ 视频播放总量），我是能理解的。

说到以内容为中心还是以创作者，想起订阅号信息列表改版和最近信息列表顺序改动。之前改版目的是提升阅读效率。站在我的角度，这些改动弱化了创作者，强调了内容。最后反倒是打乱了阅读优先级。对我个人来说适得其反。

体验槽点

「拼多多」App 评价自动跳转拍摄的设计

现在拼多多 App 上，用户点击“确认收货”，直接 Toast 提示用户“已确认收货，前往评价拍摄”然后调系统拍摄。让人反感的设计。

让商品有更多有质量的评价，是拼多多建立产品信任度的一种方式。无论是在商家侧还是用户侧，官方都在强调评价的重要性（拼多多在商家侧，在官方各种活动报名、流量分发规则中，都对商品评价有要求）。对比淘宝翻看了一些快销品类，商品评价中“有图评价”数量比例，拼多多商品占比几乎和淘宝相当。这还是在拼多多没有微淘、买家秀这些内容产品情况下。所以 C 端这种设计，数据肯定是好看。

但是拼多多 App，包括拼多多小程序内各种弹窗活动、游戏的过多设计。感觉就像是：在超市逛，刚要弯腰看商品，被一个导购急冲冲拦住塞各种营销卡片。

新功能，或者针对新功能的遮照引导

这两年遮照 + 高亮按钮的引导风格渐渐少了些。但还是时不时会碰到个别应用在使用。这种方式存续这么久，说实在的。多数原因是因为做这个项目的业务人员没有看转化数据，或者没找办法去量化引导的价值。单就走完引导流程来说，几年前我分别通过几个方法验证过（用户访谈、观察用户实际操作、真实转化数据），效果很差。特别是图中这种活动流程的引导设计，效果更差（因为这种项目往往是短周期，需求 + 研发都比较仓促，所以做成这种引导开发成本相对较低）。

建议引导用符号标记或者符号 + 提示标记就行。

产品期待

希望针对公共区域洗手间的坐便器，都能附上一次性马桶垫

每次使用马桶，就会和《爱情公寓》曾小贤一样，使用卫生纸擦干净接触面并且做好隔离。于是冒出了希望有一次性纸垫的想法。于是顺手淘宝搜索了下，果不其然是有这种产品。现在的做法只能个人备好，还是期望能公共场所能统一采购规划。

看到一些场所（上海和义乌车站）洗手间已经有单个洗手间是否为空电子屏显示了，但是我觉得这种马桶垫反而对用户价值更大。

希望互联网上能出现一款好的、长期的 RSS 产品

即刻 App 重新上线，但是以前 RSS 订阅功能全部砍掉，有些失望。每天信息都是排山倒海过来，做好聚合和选择太重要了。我已经换了很多类似产品，用过的产品包括 WeReadr、微广场、己思、微信读书（订阅公众号，但是 Web 上不显示）、Feedly，Inoreader，不是倒闭了就是加载太慢，并且还缺少 RSS 源。目前解决方案是：使用 Reeder + WeRss + RSSHub，使用一些开源服务，然后花钱买一些 RSS 源。目前还能凑合用。

其实很多人都对 RSS 订阅有需求，但是受限于商业因素，目前这种产品只能是灰产一样存在。期望大厂合作出一款 RSS 产品，虽然目前基本看不到希望。

期望一款产品拥有：

  • 聚合播客、博客、主流资讯网站、B站、主流网文网站、知乎专栏、微信订阅号（有的话就超出预期）。视频类；
  • 信息及时；
  • 不会有内容周期限制，根据个人喜好保存；
  • 附带笔记应用部分功能（分类、划线、编辑、收藏等）；
  • Web 端/客户端体验。

其实，更多的是体验上。
