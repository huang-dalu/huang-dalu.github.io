看交互设计师岗位的2个角度

2017-12-10

我觉得交互设计师在小厂几乎没有存在的必要。

第1，漫天的交互组件，电商、资讯、搜索、列表，常规产品能有的设计组件，应有尽有。交互框架越来越成熟，产品经理 + UI 做一个 70 分的交互稿并不难。你说要追求 90 分？难度很大。一方面设计这行，我认为做到 90 分，多多少少是要些天分的。而这类拥有天分的人，普遍不会去小厂，没氛围啊。另一方面，你想做到 90 分，老板可不那么想。70 分 到 90 分带来的数据效果，你确定比一个销售/运营活动更震撼吗？

第2，交互设计师这角色在团队两边不靠。又没产品经理了解业务，又不出 UI 稿。还增加沟通成本，也没上升空间。所以到最后，多半是转岗产品经理、UI。

另一个角度，交互设计师的价值，在国内又被很多不懂设计的老板给否定（概率上）。特别是B端产品。也不知道为什么一直在流传 B 端产品重逻辑，不重设计。我觉得恰恰相反，正因为B端产品逻辑复杂，交互和 UI 才更重要。因为本身操作难度就大，再设计得不好？难道都配实施人员？

说到这里，为什么很多B端产品宁可配实施，也没想着在交互和UI上下狠功夫？因为不好量化。量化有 2 个前提：

  1. 老板、交互设计师，都有数据化经营意识；
  2. 有量化的能力和配套工具。比如A/B Test系统。

你看，要求还挺高。搞到最后还不如实施来得痛快。
